Version 1.1.1 (2011-08-16)

New Features

  • Added new lobbyist portraits.

Changes

  • Changed labels in advisor window to be more descriptive.
  • Gave Question and Task areas of advisor window custom colors, to allow for easy differentiation.

Bugs fixed

  • Fixed question notification bug
  • Fixed bug that prevented forum activity.
  • Fixed bug that would allow votes to be submitted once admins approved the previous task.
  • Fixed a bug that would cause the Add Lobbyist button not to work correctly the first time it was clicked after application load.

Version 1.1.0 (2011-08-16)

New Features

  • Video window now plays flash videos hosted on remote sites.
  • When creating new users, admins now have the option to make the new user an admin.
  • Advisor window overhaul.
  • Vote window now contains a history of previous votes.
  • Windows now appear over the office, rather than as entirely new screens.

Changes

  • Modified method of updating notifications for forums.
  • Lobbyist history is now saved, users can view previous messages from lobbyists just like with memos, newspapers and videos.
  • Lobbyist bio was removed.
  • Major data structure rewrite in order to reduce the number of database writes necessary for new actions.
  • Game time removed.
  • All objects which require a timestamp now receive their timestamp from the server in order to provide proper synchronization. Any of these objects created while not logged in will not receive a timestamp until reaching the server for the first time.
  • Removed the "Time" tab from the admin Edit Scenario window. The Start/End game button now sends out emails to students and sets flags for the server to set an appropriate start or end time for AAR graphs.
  • Removed desk calendar.
  • Advisor windows no longer show vote history, only a widget for submitting votes.
  • The vote window has been rewritten to allow for simpler use. It also now contains a full history of a group's votes in reverse chronological order.
  • Reworked office interface for several groups.

Bugs fixed

  • Adjusted the locations of many notification icons in order to bring them more in line with actual image locations on screen.
  • Map code has been greatly simplified. As a consequence, though, it will only update when the map is opened -- any updates received while the map is open will not be shown until the next time that the window is opened.

Version 1.0.3 (2011-07-07)

New Features

  • After Action Report Utility added to allow admins an easy method of reviewing simulation statistics and timelines.
  • OS X install process has gotten an update to increase ease of use. Installation can now be accomplished simply by dragging the STANCE icon over to the Applications directory shortcut that pops up next to it.
  • Added "Preferences" to app to allow for multiple update channels. Students are recommended to stay with the (default) "stable" updates, but the curious can try the "devel" updates, which can have faster fixes for previous bugs, but may contain additional new bugs. Please do not use the "test" option unless specifically instructed to do so.
  • Added link checker to admin window to ensure that inject links are associated with the proper type of inject.
  • Added "Media state" label to video window to give students more information about video loading.
  • Added ability for admin to enable / disable Lobbyist, Video, Memo, and Newspaper injects that are not functioning correctly.
  • Added an option to the "File" menu which will let students clear their local STANCE data cache. This *only* clears game data, and does *not* require any sort of reinstall. All data will be redownloaded the next time that the user logs in to the game. This can fix some bugs with not being able to post, or seeing advisors locked when they should not be.
  • Users will now receive an email when their group completes a vote.

Changes

  • Removed the gray border around the game window.
  • Users will now be prompted to update before being prompted to log in.
  • Moved admin inject buttons and lists to own tab in Edit Scenario window in order to better allocate space.
  • The default vote duration has been increased to 24 hours from 12 hours.
  • Removed admin notification emails for questions.
  • Modified subject line of admin vote emails to allow for inbox filtering.

Bugs fixed

  • Lobbyist notifications should now work as intended.
  • Fixed a bug where vote real end dates would show up as "undefined" until reviewed by advisor. This should now read "N/A".

Version 1.0.2 (2011-06-16)

New Features

  • Players can now view the status of active votes that they have already voted on by clicking on the vote box. Votes cannot be changed once made, however, and this view only displays information.

Changes

  • Moved the admin-only group-switching tool from the main window to the edit scenario window.
  • Changed the time labels of the voting history list to more clearly describe the information in the start and end fields.
  • Decreased the size of several text fields in the advisor window.
  • Removed the "Report a Problem" option from the in-game help email. All issues should be discussed on the class forums or emailed directly to "techsupport@chds.us" .
  • Changed format of app title bar to include the name of user's current group.
  • Added the advisor name and title to the admin vote approval dialog.
  • Added the advisor name to the user voting dialog.
  • All dialog windows will now appear attached to their parent window, rather than free floating.
  • In order to prevent data errors for game objects (questions, tasks) where only one is allowed at a time, users who did not successfully connect to the server can no longer vote, submit questions, or submit tasks.
  • Admins must now have an active server connection to approve tasks or answer questions.

Bugs fixed

  • Fixed a bug where archived data from one database could be transferred to another.
  • Fixed a bug where long answers to questions in the advisor window would be unreadable. These answers should now be scroll windows.
  • Adjusted the location of several notification icons that were hidden by other images.
  • Fixed a bug where the admin window could display a group's office without being properly linked to that group.
  • New tasks should now immediately appear in the vote history window when submitted.
  • Admins should now properly receive notifications about new media injects.
  • Fixed an edge case where updates for one group could cause notifications to appear for another.
